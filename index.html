Planificación y Análisis de Requisitos

La tienda consistirá en una tienda online para vender skins de League of Legends más baratas que en el cliente de Riot.

Definición de Requisitos

Requisitos Funcionales

  • Catálogo: Se podrán ver las skins del juego organizadas por rarezas.
  • Carrito de compra: Habrá un carrito para añadir las skins que el cliente quiera y podrá eliminarlas a la hora de comprarlas.
  • Registro: El cliente podrá crear una cuenta dentro de la página e iniciar sesión.
  • Historial: Los clientes podrán ver las compras que han realizado anteriormente.
  • Métodos de pago: Los usuarios podrán elegir entre varios métodos de pago, como tarjeta de crédito, PayPal, Paysafecard, etc.

Requisitos No Funcionales

  • Usabilidad: Página simple y fácil de entender.
  • Rendimiento: La web debe funcionar sin tirones.
  • Seguridad: Manejar con seguridad los datos de los clientes para que no se filtre nada.
  • Disponibilidad: 24/7.
  • Adaptabilidad: Si sacan skins nuevas, no tardar en añadirlas a la web.
  • Mantenibilidad: La estructura de la página debe estar bien organizada para evitar problemas en futuras actualizaciones.

Diseño de la Arquitectura

  • Frontend: React, por su diseño adaptado a móvil y tablet.
  • Backend: Node.JS, para manejar las compras de los clientes y el carro.
  • Base de datos: MongoDB, para gestionar los usuarios, productos y pedidos.
  • Pagos: Stripe, para manejar tarjetas y criptomonedas.

Implementación

  • Configurar API REST para gestionar el catálogo, el carro y los pagos.
  • Crear interfaces con React.
  • Integrar las opciones de pago.

Pruebas

  • Realizar pruebas con JMeter para ver si la página aguanta a varios usuarios.
  • Testear la compra y el carro en diferentes dispositivos y navegadores.

Despliegue y Mantenimiento

  • Desplegar la web en AWS para asegurar escalabilidad.
  • Monitorear el rendimiento de la app usando CloudWatch y hacer actualizaciones periódicas para mejorar y corregir problemas.
